# Project Working Rules

Before changing this project, read both:

- `docs/project-structure-rules.md`
- `docs/current-handoff.md`

This project should keep new work organized by game stage:

- Main lobby
- Ready room
- Match / gameplay
- Result screen
- Shared UI
- Shared rules
- Server-only game rules

When adding or changing features, place the work in the matching module instead of adding more logic to a large page file. If a new feature changes the project structure, update `docs/project-structure-rules.md` in the same work pass so future conversations can continue from the latest rules.
