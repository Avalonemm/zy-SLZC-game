# 富饶之城 Demo 部署测试说明

当前目标：把项目发布到公网，让你和朋友可以用浏览器打开链接，创建房间、加入房间、添加测试人机，并打一局测试版。

代码仓库：

```text
https://github.com/Avalonemm/zy-SLZC-game
```

## 部署顺序

推荐顺序是：

1. 先部署后端到 Render，得到后端地址。
2. 再部署前端到 Vercel，并填写后端地址。
3. 最后回到 Render，把前端地址填给后端。

原因是前端需要知道后端在哪里，后端也需要允许这个前端连接 Socket。

## 第一步：部署后端 Render

打开这个链接：

```text
https://dashboard.render.com/blueprint/new?repo=https://github.com/Avalonemm/zy-SLZC-game
```

如果 Render 要你登录或授权 GitHub，就按页面提示登录。

页面里看到 Blueprint / render.yaml 后，使用默认配置即可。项目会创建一个后端服务：

```text
zy-board-game-server
```

如果 Render 要求填写环境变量，先填写：

```text
CLIENT_ORIGIN=http://localhost:5173
```

这是临时值。等前端 Vercel 部署完成后，再回来改成真正的前端网址。

Render 部署完成后，你会得到一个后端地址，类似：

```text
https://zy-board-game-server.onrender.com
```

打开健康检查地址：

```text
https://zy-board-game-server.onrender.com/health
```

看到 `ok: true` 就说明后端启动成功。

## 第二步：部署前端 Vercel

打开这个链接：

```text
https://vercel.com/new/clone?repository-url=https://github.com/Avalonemm/zy-SLZC-game
```

如果 Vercel 要你登录或授权 GitHub，就按页面提示登录。

项目设置保持默认即可，因为仓库里已经有 `vercel.json`。

在 Vercel 的 Environment Variables 里添加：

```text
VITE_SERVER_URL=https://你的 Render 后端地址
```

示例：

```text
VITE_SERVER_URL=https://zy-board-game-server.onrender.com
```

然后点击 Deploy。

Vercel 部署完成后，你会得到一个前端地址，类似：

```text
https://zy-slzc-game.vercel.app
```

## 第三步：回到 Render 更新前端来源

回到 Render 的后端服务 `zy-board-game-server`。

找到 Environment，修改：

```text
CLIENT_ORIGIN=https://你的 Vercel 前端地址
```

示例：

```text
CLIENT_ORIGIN=https://zy-slzc-game.vercel.app
```

保存后 Render 会自动重新部署一次。

重新部署完成后，打开 Vercel 前端地址，右下角 Socket 状态应该显示“已连接”。

## 单人公网联机测试

你一个人也可以测试，不需要开四个真人窗口。

推荐流程：

1. 打开 Vercel 前端地址。
2. 创建房间。
3. 点击“添加测试人机”，每次只添加 1 个。
4. 添加到 2 到 4 人都可以测试。
5. 非房主玩家需要准备时，人机会自动准备。
6. 房主点击开始游戏。
7. 进行角色选择、拿金币、抽牌、建造、使用技能、结束回合。
8. 刷新页面，确认可以自动回到房间。
9. 打到结算，确认排名和分数显示。

## 当前限制

- 房间数据存在 Render 后端内存里，后端重启会清空房间。
- 暂时没有数据库。
- 暂时没有账号系统。
- 当前 UI 是测试用轻量界面，正式美术和动画后置。
- Render 免费实例可能会休眠，第一次打开可能要等几十秒。

## 本地验证命令

```bash
powershell -ExecutionPolicy Bypass -File scripts/check-deploy.ps1
```
