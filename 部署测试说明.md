# 富饶之城 Demo 部署测试说明

当前阶段目标是把轻量可测版发布到公网，让朋友能打开链接创建房间、加入房间并完整打一局。

## 推荐部署组合

- 前端：Vercel
- 后端：Render Web Service
- 数据：当前仍是服务器内存，不使用数据库

## 后端 Render

1. 将项目推送到 GitHub / GitLab / Bitbucket。
2. 在 Render 中创建 Blueprint，选择仓库根目录的 `render.yaml`。
3. 创建服务后，在环境变量中填写：
   - `CLIENT_ORIGIN=https://你的前端域名`
4. 后端健康检查地址：
   - `https://你的后端域名/health`

Render 会使用：

```bash
npm ci && npm run build --workspace server
npm run start --workspace server
```

后端必须使用平台提供的 `PORT`，代码已支持 `process.env.PORT`。

## 前端 Vercel

1. 在 Vercel 中导入同一个仓库。
2. 使用仓库根目录部署，项目会读取根目录 `vercel.json`。
3. 在 Vercel 环境变量中填写：
   - `VITE_SERVER_URL=https://你的后端域名`
4. 部署后获得前端域名，再回到 Render 更新：
   - `CLIENT_ORIGIN=https://你的前端域名`

前端会使用：

```bash
npm ci
npm run build --workspace client
```

输出目录是：

```text
client/dist
```

## 公网测试清单

1. 打开前端页面，确认 Socket 状态为“已连接”。
2. 玩家 A 创建房间。
3. 玩家 B / C / D 使用房间号加入。
4. 非房主点击准备。
5. 房主开始游戏。
6. 完成角色选择。
7. 按当前行动玩家执行拿金币、抽牌、建造、技能、结束回合。
8. 刷新页面，确认可以自动恢复房间。
9. 打到结算，确认排名和分数显示。

## 当前限制

- 房间数据在服务器内存中，后端重启会清空房间。
- 暂时没有账号系统。
- 暂时没有数据库。
- 阶段 8 的正式卡牌美术和动画已后置。
- 如果 Render 免费实例休眠，第一次连接可能会有冷启动等待。
