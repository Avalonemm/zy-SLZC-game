大厅到准备界面 UI 拆分说明
版本：v0.3
最后更新：2026-07-01

目标：
本文记录从主大厅到房间准备界面的 UI 部件拆分方式。后续替换美术素材时，优先替换对应部件，不把整张界面截图当作唯一背景。

一、文字渲染规则
1. 固定 UI 文字尽量使用图片素材里的艺术字。
   例如：创建房间、加入房间、开始游戏、离开房间、准备、取消准备、已准备、未准备、设置、公告、帮助。
2. 动态内容仍然使用代码文字。
   例如：玩家名称、UID、房间号、人数、Socket 状态、输入框真实输入内容。
3. 设置、公告、帮助弹窗内的正文使用普通黑体 / 系统无衬线字体，不做艺术化字体。

二、当前资产原则
1. 不再从整张 UI 参考图里按固定坐标裁剪控件。
2. 原因：参考图是展示稿，带背景、光效、文字和相邻元素，直接裁剪会产生脏边、错位、残留文字。
3. 新方案：使用独立生成或独立绘制的 PNG 部件进行装配。
4. 动态输入类控件优先使用“空白框版本”，由代码渲染真实输入内容；带占位字版本只作为预览或特殊静态展示使用。

五、房间人数和测试人机规则
1. 当前测试版支持 2-4 人开局。
2. 房间 maxPlayers 固定为 4，但字段保留，后续可扩展到 2-8 人。
3. 添加测试人机每点一次只添加 1 个。
4. 房主退出后，服务端自动把房主移交给玩家列表中的下一位玩家。
5. 新房主自动 isHost = true 且 isReady = true。
